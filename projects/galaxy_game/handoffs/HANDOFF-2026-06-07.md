## Key Architecture Decisions Made This Session

### Cycler Network Architecture
Cyclers are **mobile space stations** on continuous orbital arcs — not point-to-point transports. Smaller craft match velocity and dock briefly to transfer cargo/crew.

**Core Sol cycler route:**
```
L1 → Mars (Phobos/Deimos depot) → Venus (orbital processing stations) → L1
```

**Outer system:** Mars ↔ Saturn/Titan resource route — flexible, AI Manager optimizes

**AI Manager cycler decisions:**
- Add cycler when settlement traffic justifies cost
- Reroute when settlement matures or is abandoned  
- Abandon when route no longer economically viable
- Emergency fast ship from L1 when cycler schedule can't meet critical shortage

**Consumption formula for non-Luna worlds:**
```
order_quantity = daily_consumption × days_to_next_cycler_pass × margin_factor
+ cycler_buffer_capacity (if cycler holds emergency reserve)
```

### Station Architecture by World
- **L1** — central hub, shipyard, depot. Everything flows through here
- **Mars** — Phobos/Deimos as natural orbital depots, low gravity well, easy staging
- **Venus** — orbital processing stations only (surface hostile). Skimmers harvest atmosphere, return to orbital stations. More factory than settlement.
- **Belt** — mining operations, served by cycler loop

### Supply Chain Evolution
```
Act 1 early:  Earth → Luna (AstroLift HLT, 150 GCC/kg)
              Earth → Venus/Titan (skimmer deployment)
              Venus/Titan skimmers → Luna (exotic gases)

Act 1 mature: Luna ISRU → L1 Station
              Earth launches → L1 Station
              L1 → Cyclers → Mars/Venus/Belt
              (Earth import cost drops as L1/Luna mature)

Act 2+:       Cycler network self-sustaining
              Earth dependency minimal
              Portal gates reduce intra-system transit
```

### Venus Skimmer Evolution
Early game: Skimmers return to Luna (before L1 exists)
Once L1 + cycler operational: Venus orbital stations become skimmer destination, transfer to cycler, delivered system-wide

## Architecture Context (Accumulated This Session)

### Location Model (Two-Tier)
- `SpatialLocation` — x/y/z coordinates, body position in system space. Has `distance_to`. Use for inter-body distances.
- `CelestialLocation` — lat/lon + altitude, position ON a body. Use for settlement/gate positions.
- Orbital simulation does not exist yet — positions are static seeded coordinates

### Precursor Mission Philosophy
Luna precursor is fully robotic — no humans, no life support consumption during build phases:
1. Power & Comms
2. Resource Harvesting / ISRU bootstrap
3. 3D printing components, tank farm, lava tube prep

5. Human arrival gate (only when habitat verified, power stable, consumables stockpiled)
