## Context

The `Deposit::PlausibilityEngine` is a world-neutral geological evaluation service. When a survey action triggers on a feature, the engine evaluates what deposits are plausible for that feature given the celestial body's properties. It returns spawn parameters — probability, mass range, equipment tier — that the `DepositSpawner` uses to create `ResourceDeposit` records.

**The engine is world-neutral by design.** It never hardcodes Luna, Mars, or any specific world. All world knowledge comes from the celestial body's stored properties and the rules config JSON.

**Three tiers of world data the engine handles:**
- **Known worlds** (Luna, Mars, Europa) — real scientific data in JSON files, rich feature data
- **Partial worlds** — catalog-level data, estimated values, incomplete features
- **Generated worlds** — no JSON files, synthetic properties generated by `PrecursorCapabilityService` on first visit

**The engine never crashes on missing data.** It degrades gracefully:
- Data exists → use it
- Data partial → use what exists, apply conservative body-type defaults for gaps
- Data absent → generate believable values from body-type archetypes

**Dynamic world rule** — new craters can form, ice can deplete, volcanic activity can create new features. The engine never caches deposit absence. A re-survey with better equipment may find what a previous survey missed.

**Relevant Architecture Docs** — read before starting:
- `docs/new_agent/rules/DECISIONS.md` — locked architectural decisions including lazy spawn decisions
- `docs/new_agent/rules/GUARDRAILS.md` — execution rules
- `docs/mission_profiles/LUNA_BASE_ESTABLISHMENT.md` — Luna resource hierarchy and tier definitions

## Service Interface

**Location**: `app/services/deposit/plausibility_engine.rb`

```ruby
module Deposit
  class PlausibilityEngine
    # Main entry point
    # Returns hash of viable deposit types with spawn parameters
    # Never raises — returns empty hash if nothing viable
    def self.evaluate(feature, celestial_body)
      new(feature, celestial_body).evaluate
    end

    def initialize(feature, celestial_body)
      @feature = feature
      @body = celestial_body
      @rules = load_rules
      @body_props = resolve_body_properties
    end

    def evaluate
      viable = {}
      @rules['deposit_plausibility_rules'].each do |deposit_type, rule|
        result = evaluate_deposit_type(deposit_type, rule)
        viable[deposit_type] = result if result[:spawn_probability] > 0.0
      end
      viable
    end
  end
end
```

**Output format per deposit type:**
```ruby
{
  'water_ice' => {

    mass_fraction: 0.023,           # rolled from budget fraction range

}
```

## Feature Classification

The engine classifies features using available data with graceful fallback:

```ruby
def classify_feature(feature, rule)
  return :default unless rule['preferred_feature_types']
                           .include?(feature.feature_type)

  attrs = feature.attributes || {}
  lat = feature.coordinates&.dig('latitude').to_f.abs

  if feature.feature_type == 'crater'
    if attrs['permanently_shadowed'] == true
      :permanently_shadowed_crater
    elsif lat >= 70
      :polar_crater
    elsif lat >= 50
      :polar_crater
    else
      :equatorial_crater
    end
  elsif feature.feature_type == 'lava_tube'
    # Check if body has active volcanism
    if @body_props[:volcanically_active]
      :volcanic_lava_tube
    else
      :default_lava_tube
    end
  else
    :default
  end
end
```

## Body Property Resolution

Resolves body properties from available data with graceful fallback:

```ruby
def resolve_body_properties
  props = {}

  # stored_volatiles — use real data or archetype default
  props[:stored_volatiles] = @body.stored_volatiles.presence ||
    archetype_default(:default_stored_volatiles)

  # materials — use real data or archetype default
  props[:materials] = @body.materials.map(&:material_name).presence ||
    archetype_default(:default_materials)

  # atmosphere — use real data or archetype default
  props[:atmosphere_pressure_atm] = @body.atmosphere_pressure_atm.presence ||
    archetype_default(:default_atmosphere_pressure_atm)

  # temperature — use real data or archetype default
  props[:average_temperature_k] = @body.average_temperature_k.presence ||
    archetype_default(:default_temperature_k)

  props
rescue => e
  Rails.logger.warn("PlausibilityEngine: body property resolution failed — #{e.message}")
  archetype_defaults_for_unknown_body
end
```
