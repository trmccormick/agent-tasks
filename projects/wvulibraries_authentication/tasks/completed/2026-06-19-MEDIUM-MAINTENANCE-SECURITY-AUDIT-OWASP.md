# Task: PHP Security Audit (OWASP Top 10 Assessment)

## Problem Statement & Investigation Findings

### Temp Account Lifecycle Discovery (CRITICAL CONTEXT)

**ACTUAL SYSTEM DESIGN** (discovered during investigation):
- ✅ **74 pre-generated static accounts** in `tempAccounts` table (lib-temp-001 through lib-temp-074)
- ✅ **No dynamic creation logic exists** - Accounts loaded from SQL dump, never created at runtime
- ✅ **Passwords stored PLAIN TEXT** (varchar(12)) - e.g., `9&Pqp+4N`, `$9UaR%2j`
- ✅ **No expiration/cleanup mechanism found** - No timestamps, no daily reset cron jobs
- ⚠️ **passwordReset.php EXISTS but is UNNECESSARY** - System uses static pool, not user-initiated resets

**User Workflow (inferred from code)**:
1. Librarians view temp accounts via `/authentication/tempAccounts/index.php`
2. Accounts distributed to patrons for one-time library system access  
3. **System likely manually regenerates passwords periodically** (no automated reset found)
4. Temp accounts authenticate against WVU-AD LDAP using plain text password bind

### Security Issues Identified:

1. **CRITICAL: SQL Injection Vulnerability** - String interpolation in EngineAPI queries (Seq 3 Phase 2 deferred)
2. **HIGH: Plain Text Password Storage** - Acceptable risk for ephemeral accounts, but should document justification
3. **MEDIUM: LDAP Bind with Plaintext** - Required by protocol, but passwords stored unencrypted in DB
4. **LOW: Missing Account Expiration** - No automated cleanup (may be intentional design)

This audit addresses all four issues with fixes and documentation.

### Task 4: String Interpolation → Prepared Statements (Seq 3 Phase 2)

**Current State** (from Seq 3):
- EngineAPI 3.2 migrated from `mysql_*` → `mysqli_*`
- But still uses string interpolation instead of prepared statements
- Example: `vsprintf("SELECT * FROM users WHERE id=%d", $id)` (vulnerable)

**Files Requiring Hardening**:
1. Extracted engineAPI modules (in `src/phpincludes/engineAPI/engine/`)
2. Authentication-specific:
   - `src/phpincludes/databaseConnectors/database.lib.wvu.edu.remote.php`
   - `src/public_html/engineIncludes/login-3.0.php`
   - `src/public_html/authentication/acl.php`
   - `src/public_html/authentication/tempAccounts/*.php`

**Refactoring Pattern**:
```php
// BEFORE (vulnerable)
$query = sprintf("SELECT * FROM users WHERE id=%d AND name='%s'", $id, $name);
$result = $mysqli->query($query);

// AFTER (secure)
$stmt = $mysqli->prepare("SELECT * FROM users WHERE id = ? AND name = ?");
$stmt->bind_param("is", $id, $name);
$stmt->execute();
$result = $stmt->get_result();
```

Test expectations:
- ✅ All string interpolation queries converted to prepared statements
- ✅ Zero SQL injection vulnerabilities in codebase review
- ✅ All 70 tests pass after refactoring
- ✅ Performance verified (prepared statements may slightly improve query reuse)
